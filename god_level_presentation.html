The Stock Market

A global platform for buying & selling company shares.
Prices fluctuate based on performance, news, and world events.

Real-Life Example

If positive news breaks about Apple launching a new iPhone, investors rush to buy. The demand increases, and the stock price goes up. Our system is built to track this exact news-to-price relationship.

"To start, what is the stock market? It's simply where shares are traded. Prices go up and down mostly because of news. For example, if Apple announces a great product, its price shoots up. My project tracks this data."

The Problem

Information Overload

Thousands of financial articles drop daily. Humans physically cannot process all this data.

Emotional Trading

Retail investors panic-sell or FOMO-buy based on a single headline.

Fragmented Tools

Users juggle 5 different apps just to see charts and read related news.

"The main problem we face is Information Overload. There is too much news. Second, people trade on emotions and panic. Third, users have to open multiple apps to see charts and news separately."

Project Objective

Build a single unified platform for live stock prices and news.

Use Artificial Intelligence to scan articles and declare a Positive/Negative score.

Help users make safe, data-driven decisions without emotional bias.

"Our objective is to build ONE single platform. We want to use AI to automatically read the news and give it a Good or Bad score. This will help normal people make safe trading decisions."

Key Features

Global Search: Instantly find any stock.
Real-Time Data: Live, millisecond-accurate prices.
Interactive Charts: Visual 30-day market trends.
AI Sentiment: Auto-calculates market mood.
Live vs Demo: Toggle to practice without real risk.
"The features of my project are: A search bar for any stock. Real-time live pricing. 30-day interactive history charts. AI sentiment calculation. And a toggle switch between Demo practice mode and Live mode."

Tech Stack

Frontend

React.js (UI Library) Tailwind CSS (Glassmorphism styling) Recharts (Financial graphs)

Backend

Node.js & Express (Server logic) REST APIs (Data transfer) Render (Cloud hosting)
"For the user interface, I used React.js and Tailwind CSS to make it look beautiful and modern. For the backend server, I used Node.js and Express to securely handle all the data."

API Integration

We don't create data; we fetch it from trusted global sources.

Yahoo Finance & Finnhub

Fetches the live stock prices and history charts.

NewsAPI

Fetches the top latest news articles about the company.

Smart Fallback System

If one API server crashes, the system instantly switches to another to prevent errors.

"Since we don't own stock data, we use APIs. Yahoo Finance and Finnhub provide the prices. NewsAPI provides the articles. And we built a fallback system so if one API fails, the website still runs."

System Architecture

React UI

User Search

Node.js

Process Data

External APIs

Yahoo / News

Data is processed by AI and returned as JSON to the UI.

"The architecture is straightforward. The user searches on the React UI. The request goes to the Node.js server. Node fetches data from external APIs, calculates the AI score, and sends the final result back to the user."

Working (Step-by-Step)

1. User opens the web application.
2. Types a company name in the search bar (e.g. "Google").
3. System securely fetches live prices and recent news in the background.
4. The AI Engine scans the article text and calculates a sentiment score.
5. Dashboard updates instantly with charts and the final verdict.
"Step by step: The user opens the app, searches 'Google'. The backend quietly fetches prices and news. The AI reads the news, generates a score, and the dashboard updates instantly."
